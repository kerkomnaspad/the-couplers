Bad Code Smell : The Couplers

Middle Man

Class yang menjadi perantara dengan class lain, tanpa memberikan fungsionalitas.


Contoh code:

public class Customer {
  private CustomerService customerService;

  // Constructor untuk membuat instance CustomerService
  public Customer() {
   this.customerService = new CustomerService();
  }

  // Method untuk membuat customer baru
  public void createCustomer(String name, String email) {
   customerService.createCustomer(name, email);
  }

  // Method untuk mengupdate informasi customer
  public void updateCustomer(String name, String email) {
   customerService.updateCustomer(name, email);
  }

  // Method untuk menghapus customer
  public void deleteCustomer(String name) {
   customerService.deleteCustomer(name);
  }
}


Kode diatas adalah class Customer yang digunakan hanya mengakses class customerService. Cara yang lebih baik adalah dengan mengakses customerService secara langsung tanpa melalui class Customer



Setelah refactor:

// Membuat instance CustomerService
CustomerService customerService = new CustomerService();

// Membuat pelanggan baru
customerService.createCustomer("John Doe", "john@example.com");

// Memperbarui informasi pelanggan
customerService.updateCustomer("John Doe", "john.doe@example.com");

// Menghapus pelanggan
customerService.deleteCustomer("John Doe");

Solusi: Mengakses class yang diperlukan secara langsung tanpa melalui class perantara.
