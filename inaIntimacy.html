Bad Code Smell : The Couplers

Inappropriate Intimacy

Code yang menggunakan method atau field pada class lain. Umumnya terlihat pada class yang sering berhubungan dan bergantung.


Contoh code:

public class Person {
  private String name;
  private Address address;


  public Person(String name, Address address)
 {
  this.name = name;
  this.address = address;
  }


 public String getName()
 {
  return name;
 }


 public Address getAddress()
 {
  return address;
 }


 public String getAddressCity()
 {
  return address.getCity();
 }


 public void setAddressCity(String city)
 {
  address.setCity(city);
 }


}


public class Address {
 private String city;


 public Address(String city) {
  this.city = city;
 }


 public String getCity() {
  return city;
 }


 public void setCity(String city) {
  this.city = city;
 }


}


Pada code diatas, class Person mengakses class Address dan juga dapat mengakses fieldnya, yang melanggar prinsip enkapsulasi



Setelah refactor:

public class Person {
  private String name;
  private Address address;


  public Person(String name, Address address)
 {
  this.name = name;
  this.address = address;
  }


 public String getName()
 {
  return name;
 }


 public Address getAddress()
 {
  return address;
 }


}


public class Address {
 private String city;


 public Address(String city) {
  this.city = city;
 }


 public String getCity() {
  return city;
 }


 public void setCity(String city) {
  this.city = city;
 }


}

Solusi: Melakukan move method dan move field untuk memindahkan bagian dari suatu class ke class yang menggunakan method tersebut.
